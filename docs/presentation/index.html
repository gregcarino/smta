SMTA

Background

Transport companies sign insurance policies with insurance companies to cover the risk that goods are damaged during transportation.

These insurance policies generally enforce certain conditions (route, schedule, etc.).

When the goods are damaged, the transport company opens up a claim.

The insurance company then checks if the conditions of the policy were respected.

There are several gaps in the this process:

It is very difficult for the insurance company to determine if the policy conditions were fulfilled.

The premiums do not always accurately reflect the risks of a policy. Mostly because of the lack of first-hand data.

SMTA

Goals

Create a platform which allows users to manage shipments, tracking devices and sensors.

Create tracking devices (prototypes) for collecting the data from transports.

Enable real-time tracking and notification capabilities.

Use the data collected to expose statistics and analytic scenarios to aid insurance companies in determining the premium ammounts.

Overview

Technologies

Two device prototypes were made: an Arduino board and an Android application.

The platform for collecting data, monitoring, tracking, etc. was build with SAP Cloud Platform.

All user interfaces were created with SAP UI5.

HANA

A HANA (DB, XSC) sits at the center of the solution.

Conceptually, it is split into three components: Administration, Tracking, Analytics.

The RESTful services are exposed via two APIs:

  • Primary API (for data collection / mgmt.)
  • Secondary API (for analytic scenarios)

SMTA

Web Apps

All the cloud web apps are deployed to the Fiori Launchpad.

They are grouped based on the underlying HANA component (Administration, Tracking, Analytics).

Administration Apps

Responsible for managing the transports, devices, sensors, routes, schedules, organizations and people.

We can define conversion formulas and virtual sensors.

For each transport, limits can be defined. When a limit is breached, a notification email is sent.

Tracking Apps

The overview app shows in real time the evolution of all active transports and allows scrolling through time to see the previous state of the transports.

The trace can be used to see an animation of the transport evolution.

Analytics Apps

Cover several use cases:

  • Viewing the evolution of a single transport.
  • Comparing several transports against each other.
  • Generating system-wide statistics.
  • Performing simple forecasts (SAP HANA PAL).

Devices

Android App

Uses the built-in GPS and sensors.

Can either store the collected data locally and upload it all at once or send it to the cloud as soon as it is read.

Arduino Board

Connected to serially-commanded GPS and Bluetooth modules.

It communicates with a Bluetooth enabled device (a laptop) which forwards data to the cloud (with the help of a Proxy C# application).
